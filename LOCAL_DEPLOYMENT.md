# 🎯 Alice Chess - Локальное развертывание

## 🚀 Быстрый запуск

### Вариант 1: Полностью автоматический запуск (рекомендуется)
```bash
./start_local.sh
```
Эта команда автоматически:
- Запускает локальный Stockfish сервер (chessapi)
- Запускает Alice Chess Flask сервер
- Настраивает все переменные окружения

### Вариант 2: С кастомными настройками API
```bash
# Использовать локальный сервер (по умолчанию)
./start_local.sh

# Использовать удаленный сервер alice-chess.ru
./start_local.sh --remote

# Использовать свой сервер
./start_local.sh --url "https://my-chess-server.com/bestmove/" --key "my-api-key"

# Показать справку
./start_local.sh --help
```

### Вариант 3: Ручной запуск

1. **Запустите Stockfish сервер:**
   ```bash
   # Перейдите в директорию chessapi (настройте путь через переменную CHESSAPI_DIR)
   cd ${CHESSAPI_DIR:-../chessapi}
   # Активируйте виртуальное окружение (настройте через CHESSAPI_VENV)
   source ${CHESSAPI_VENV:-chessapi_env}/bin/activate  # или Scripts/activate на Windows
   python -m uvicorn chessapi:app --host 0.0.0.0 --port 8000
   ```

2. **В новом терминале запустите Alice Chess:**
   ```bash
   # Перейдите в директорию проекта Alice Chess
   cd ${PROJECT_ROOT:-.}
   source env/bin/activate  # или Scripts/activate на Windows
   export CHESS_API_URL="http://localhost:8000/bestmove/"
   export CHESS_API_KEY="your-api-key-here"
   python play_local.py
   ```

## 🎮 Как играть

После запуска `play_local.py` вы увидите интерактивное меню. Введите команды на русском языке:

### Основные команды:
- `Давай сыграем в шахматы` - начать новую игру
- `Белые` или `Черные` - выбрать цвет фигур
- **Шахматные ходы:**
  - `е2е4` - пешка e2-e4
  - `Кf3` - конь на f3
  - `О-О` - короткая рокировка
  - `О-О-О` - длинная рокировка
  - `Лd5` - ладья на d5 (кириллица)
- `Покажи доску` - показать текущую позицию
- `Новая игра` - начать новую партию
- `Помощь` - показать справку
- `Сдаюсь` - сдаться
- `Ничья` - предложить ничью
- `Уровень сложности` - изменить уровень игры (1-20)
- `выход` или `quit` - выйти из игры

## 🛠 Технические детали

### Архитектура:
- **alice_chess.py** - Основная логика навыка
- **game.py** - Управление шахматной партией
- **play_local.py** - Интерактивный клиент для игры

### Зависимости:
- Python 3.9+
- Flask - веб-сервер
- python-chess - шахматная библиотека
- requests - HTTP-клиент

### API:
Сервер доступен по адресу `http://localhost:5000` и принимает POST-запросы в формате Alice Skills API.

## 🐛 Устранение неполадок

### Сервер не запускается:
```bash
# Проверьте виртуальное окружение
source env/bin/activate
python -c "import alice_chess; print('OK')"
```

### Зависимости не установлены:
```bash
source env/bin/activate
pip install -r requirements.txt
```

### Игра не отвечает:
Проверьте, что сервер работает:
```bash
curl http://localhost:5000/health
```

## 🎯 Особенности

- **Шахматный движок:** Полноценный Stockfish 17 с локальным сервером
- **Гибкая конфигурация:** Поддержка локального и удаленного API серверов
- **Профессиональный ИИ:** Настоящий шахматный движок без fallback
- **API аутентификация:** Защищенный доступ с API ключами
- **Поддержка языков:** Русский и английский (алгебраическая нотация + кириллица)
- **Уровни сложности:** 1-20 (влияет на время и глубину анализа)
- **Сохранение состояния:** Между запросами в рамках сессии
- **Специальные ходы:** Рокировка, превращение пешек, взятие на проходе

## ⚙️ Конфигурация

Alice Chess поддерживает гибкую настройку через переменные окружения.

### Переменные окружения для путей:

- **`PROJECT_ROOT`**: Путь к корневой директории проекта Alice Chess
  - По умолчанию: текущая директория (`.`)

- **`CHESSAPI_DIR`**: Путь к директории Chess API проекта
  - По умолчанию: `../chessapi` (родительская директория)

- **`CHESSAPI_VENV`**: Имя виртуального окружения Chess API
  - По умолчанию: `chessapi_env`

### Примеры настройки путей:

```bash
# Если проекты находятся в разных директориях
export PROJECT_ROOT="/path/to/alice-chess"
export CHESSAPI_DIR="/path/to/chessapi"
export CHESSAPI_VENV="venv"

# Запуск с кастомными путями
./start_local.sh
```

## ⚙️ Конфигурация Chess API

Alice Chess поддерживает гибкую настройку Chess API через переменные окружения, которые можно хранить в `.env` файле (Laravel-style).

### Переменные окружения:

- **`CHESS_API_URL`**: URL сервера с Chess API
  - По умолчанию: `http://localhost:8000/bestmove/`
  - Для удаленного: `https://alice-chess.ru:8000/bestmove/`

- **`CHESS_API_KEY`**: API ключ для аутентификации
  - Для локального сервера: предустановленный ключ
  - Для удаленного сервера: требуется валидный ключ

### Настройка через .env файл (рекомендуется):

1. **Скопируйте шаблон:**
   ```bash
   cp .env.example .env
   ```

2. **Отредактируйте `.env` файл:**
   ```bash
   # Alice Chess Environment Variables

   # Пути к проектам (опционально)
   # PROJECT_ROOT=/path/to/alice-chess
   # CHESSAPI_DIR=/path/to/chessapi
   # CHESSAPI_VENV=venv

   # Chess API настройки
   CHESS_API_URL=https://alice-chess.ru:8000/bestmove/
   CHESS_API_KEY=your-api-key-here
   ```

3. **Запустите с автоматической загрузкой:**
   ```bash
   ./start_local.sh  # Автоматически загрузит .env
   ```

### Альтернативная настройка через переменные окружения:

```bash
# Локальный сервер
export CHESS_API_URL="http://localhost:8000/bestmove/"
export CHESS_API_KEY="your-api-key-here"

# Удаленный сервер (если есть ключ)
export CHESS_API_URL="https://alice-chess.ru:8000/bestmove/"
export CHESS_API_KEY="your-valid-api-key"

# Запуск игры
python play_local.py
```

### 🔒 Безопасность:

- `.env` файл добавлен в `.gitignore` и не будет committed в git
- API ключи хранятся локально и не попадают в репозиторий
- Используйте `.env.example` как шаблон для других разработчиков
